Approach

To solve this, it is necessairy to keep track of the current substring being read by the loop. For that, you can use the iterator of the for and another integer variable, but I optioned to use an auxiliar string variable (currentSubstring) and use a ranged for. IMO, it’s better for reading. I think the other option it’s memory wisier. Whatever. Continuing: whenever the solution find a character that already is in the substring, it saves the longest substring and keep the forward part of the current substring where there isn’t an invalid character. Example: “dvdf”; After found the “dv”, it’ll concat the last ‘v’ to the new substring, because the longest substring in this case it’s “vdf”; if you did not add this procedure to the algorithm, the longest substring would be “dv” or “df”, which is incorrect.

Complexity

  • Space complexity:
    Also, I think we got an O(n)O(n)O(n), guys!

Code

class Solution {
public:
    int lengthOfLongestSubstring(string str) {
        string longestSubstring = "", currentSubstring;
        for (const char character : str) {
            const int position = currentSubstring.find(character);
            if (position == -1) currentSubstring += character;
            else {
                longestSubstring = currentSubstring.length() > longestSubstring.length() ? currentSubstring : longestSubstring;
                currentSubstring = currentSubstring.substr(position + 1, currentSubstring.length()) + character;
            }
        }
        return max(longestSubstring.length(), currentSubstring.length());
    }
};
